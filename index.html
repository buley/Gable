Gable

Shadow structure for Google tables.

Gabel makes it easy to create Google data tables, which in turn make it easy to draw graphs and plot charts.

You take care of data and visualization. Gable covers the boring stuff.

Examples

1. Setup

Gable depends on the JavaScript Google Visualization API but will always take care of loading the right files. To enable Gable, simply include the script tag.

<script type='text/javascript' src='https://raw.github.com/editor/Gable/master/gable.js'></script>

Note: It's not a great idea to serve the file directly from Github on a live website, but the above script tag will get you going.

Gable has not yet been tested across all browsers.

2. Create

To create a data table, pass Gable the data and a unique string to identify it.

Gable( 'jack' ).create( [ [ 'work', .9 ], [ 'play', .1 ] ] );

Pass in a keyed object to control visualization options.

3. Draw

Pass Gable a data table's unique id, a target DOM id and visualization type to draw it on page.

Gable( 'jack' ).draw( 'dullard', 'pie' );

The above code would attempt to append a pie chart to an HTML element with the id `dullard`.

4. Update

You can update columns, rows and cells by index. Pass the `update` method a row id, column id or both to update a row, column or cell with a given value.

Gable( 'jack' ).update( { 'row': 1, 'column': 2, 'value': .8 );

Data tables are automatically wired to visualizations so any changes you make will be automatically reflected live. To manually control these updates for a particular table, use the `stage` and `commit` methods.

var table = Gable( 'jack' );
table.delay();
table.update( { row: 1, column: 2, value: .8 );
table.update( { row: 2, column: 2, value: .2 } );
table.commit();

Note: All calls to Gable are chainable so the above code could be rewritten as Gable( 'jack' ).delay().update( { row: 1, column: 2, value: .8 ).update( { row: 2, column: 2, value: .2 } ).commit();

The above code updates two rows before making a change live.
